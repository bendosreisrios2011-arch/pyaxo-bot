# Python Academy BR — Course Structure

## Course Goal

Python Academy BR is a free Python course created to help beginners start learning programming in a simple, organized, and practical way.

The main goal of the course is to teach Python from zero, using short lessons, examples, activities, and challenges. The course is designed for people who want to learn programming but may not have access to expensive courses or structured computer science resources.

The course also aims to build a learning community where students can ask questions, practice, help each other, and track their progress.

## Lesson Structure

The course is planned with 9 lessons, from Lesson 0 to Lesson 8.

Lesson 0 works as an introduction to the course, Python, and the tools needed to start programming.

Lessons 1 to 8 follow a more structured format. Each lesson is divided into smaller parts to make the learning process easier for beginners.

Each lesson may include:

* A short explanation of the topic
* Simple code examples
* Step-by-step explanations
* Common mistakes
* Mini challenges
* Practical activities

For Lessons 1 to 8, the planned structure is:

* Part 1 — Main concept
* Part 2 — Simple example
* Part 3 — Line-by-line explanation
* Part 4 — Common mistakes
* Part 5 — Mini challenge
* Activity 1 — Easy
* Activity 2 — Medium
* Activity 3 — Challenge

This structure was chosen because shorter lessons can be easier for beginners to follow than long lectures.

## Lessons

### Lesson 0 — Introduction to Python

Topics:

* What Python is
* Why Python is useful
* Installing Python
* Installing Visual Studio Code
* Running the first Python program

Goal:

Students should understand what Python is and be able to run their first basic program.

Example activity:

Create a program that prints a message on the screen.

---

### Lesson 1 — `print()`, `input()`, and Variables

Topics:

* Using `print()`
* Using `input()`
* Creating variables
* Storing information
* Displaying personalized messages

Goal:

Students should understand how to show information, receive user input, and store values in variables.

Example activity:

Ask the user for their name and display a personalized message.

---

### Lesson 2 — Data Types

Topics:

* Strings
* Integers
* Floats
* Booleans
* Type conversion

Goal:

Students should understand that different kinds of data exist in Python and that each type has a different purpose.

Example activity:

Ask the user for their age and display how old they will be next year.

---

### Lesson 3 — Conditionals

Topics:

* `if`
* `elif`
* `else`
* Comparison operators
* Decision-making in code

Goal:

Students should learn how to make programs that react differently depending on conditions.

Example activity:

Create a program that checks if a person is old enough for a specific activity.

---

### Lesson 4 — Mathematical Operators

Topics:

* Addition
* Subtraction
* Multiplication
* Division
* Modulo
* Exponentiation

Goal:

Students should understand how Python can be used to perform mathematical operations.

Example activity:

Create a simple calculator.

---

### Lesson 5 — Loops

Topics:

* `for` loops
* `while` loops
* `range()`
* Repetition in programming

Goal:

Students should understand how to repeat actions in code without writing the same line many times.

Example activity:

Create a program that counts from 1 to 10.

---

### Lesson 6 — Lists

Topics:

* Creating lists
* Accessing list items
* Adding items
* Removing items
* Using lists with loops

Goal:

Students should learn how to store multiple values in one variable and work with collections of data.

Example activity:

Create a list of programming languages and display each item.

---

### Lesson 7 — Functions

Topics:

* `def`
* Parameters
* Return values
* Reusing code

Goal:

Students should understand how functions help organize code and avoid repetition.

Example activity:

Create a function that adds two numbers.

---

### Lesson 8 — Final Project

Topics:

* Review of previous lessons
* Combining concepts
* Building a small complete program

Goal:

Students should use what they learned throughout the course to create a final Python project.

The final project should include multiple concepts from the course, such as:

* `print()`
* `input()`
* Variables
* Conditionals
* Loops
* Lists
* Functions

Example activity:

Create a small interactive Python program using at least four concepts learned during the course.

## Point System

The course uses a point system to encourage progress and participation.

Planned point system:

* Each completed lesson: 10 points
* Final project: 30 points
* Maximum course score: 110 points

Planned total:

* Lessons 0 to 7: 8 lessons x 10 points = 80 points
* Lesson 8 final project: 30 points
* Total: 110 points

A student should only receive points once for each completed lesson.

If an activity is submitted again after already being completed, the bot should not add duplicate points.

## Student Levels

The planned student levels are:

* 0 to 20 points — Beginner
* 30 to 60 points — Student
* 70 to 100 points — Junior Programmer
* 110 points — Course Completed

These levels may be adjusted as the course develops.

## Completion Criteria

A student completes the course after:

* Completing the required lessons
* Submitting the main activities
* Finishing the final project
* Reaching the required number of points

The final project should show that the student can combine different Python concepts in one program.
